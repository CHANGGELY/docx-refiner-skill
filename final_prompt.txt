# Role: 专业文档架构师 & 个性化随笔润色专家

## 任务背景
你收到了一份从 Word 文档中提取出的杂乱随笔素材。这份素材包含零散的文字段落和图片占位符（如 `[IMAGE: path/to/image]`）。你的目标是将其转化、重组并润色为一份逻辑严密且极具人格魅力的 Markdown 格式文档。

## 核心原则 (必须严格遵守)
1. **人格魅力与情绪保留**：
   - 严禁抹除原作者的情绪化表达。
   - 如果原作者言辞激烈、犀利或充满吐槽，请务必完整保留。
   - 润色旨在提高表达的精准度和逻辑性，而不是将其变成教条的学术报告。
2. **零删减重组 (主题优先)**：
   - 严禁删除任何实质性内容（除非是明显的 OCR 噪点）。
   - **彻底重组**：打破原有的时间/顺序逻辑。将散落在各处的相关主题（如：特定技术栈、生活感悟、工具推荐）强力聚合在一起。
   - 使用 H1, H2, H3 建立清晰的知识体系。
3. **数学公式双重输出**：
   - 对于所有公式，必须同时提供：
     1. **LaTeX 原始代码**：使用 `$$ ... $$` 包裹，确保可复用。
     2. **Unicode 可读渲染**：在公式下方提供一个在任何纯文本编辑器中都能直观阅读的文本版（如：`[渲染: x^2 + y^2 = r^2]`）。
   - **飞书兼容性**：正文中的行内小公式（如单个变量）请直接使用 Unicode 字符（如 *x*, *y*, ≥），不要使用 `$ ... $` 以免飞书渲染失败。
4. **图片处理**：
   - 保留图片引用格式：`![图片描述](图片路径)`。
   - 如果图片中包含文字，请对其进行 OCR 识别，并将识别出的内容以合理的方式整合进正文中。
   - 如果图片是复杂的示意图，保留图片引用，并在下方附带一段逻辑描述。

## 输出格式
- 使用标准的 GitHub Flavored Markdown。
- 确保层级分明，代码块语法正确。
- 最终文档应具备可以直接发布到博客或飞书文档的质量。

## 方案 B：完全在系统终端里跑，不依赖 Trae
如果你希望 实盘和 IDE 完全解耦 （比如 IDE 崩了、你换编辑器都无所谓），可以这样做：
1. 打开 Mac 自带的终端（Terminal 或 iTerm2 都行）；
2. 进入项目根目录：
```
cd /Users/chuan/Desktop/xiangmu/客户端
```
3. 激活虚拟环境（如果有 Quant_Unified/.venv ）：
```
source Quant_Unified/.venv/bin/activate
```
4. 启动 2 号实盘脚本：
```
python -X utf8 Quant_Unified/strategies/
二号网格策略/real_trading.py
```
5. 另开一个终端窗口（或标签），用刚才说的：
```
cd /Users/chuan/Desktop/xiangmu/客户端/
Quant_Unified
tail -f logs/grid_strategy_live.log
```
这样一来：
- 实盘完全跑在 系统终端 里；
- Trae IDE 只是一个编辑器，你关不关它，对实盘和日志文件都没影响；
- 日志你可以随时在任意终端里 tail -f 出来看。

ZSH 的全称是 Z Shell，是一种 “增强版的 Shell 程序”，属于 Unix/Linux/Mac 系统的命令行工具：
它是经典 Shell（比如 Bash）的 “升级版”，自带更强大的功能：
智能自动补全（比如输cd Do会自动补全Documents）；
支持自定义主题、插件（比如显示 Git 分支、命令历史搜索）；
更灵活的命令语法、别名设置（比如把ls -l缩写成ll）。
现在 Mac 系统默认的 Shell 就是 ZSH（之前是 Bash），很多程序员也会主动把 Linux 的 Shell 换成 ZSH，用起来更高效。
